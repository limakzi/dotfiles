# General Principles

- Follow the Boy Scout Rule: leave the code cleaner than you found it.
- Don't Repeat Yourself (DRY): every piece of knowledge should have a single, authoritative representation.
- Prefer simplicity (KISS): choose the simplest solution that works.
- You Aren't Gonna Need It (YAGNI): do not add functionality until it is necessary.
- Write code for the reader, not the compiler.
