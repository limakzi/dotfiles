# Naming

- Use intention-revealing names; names should answer why something exists and what it does.
- Avoid disinformation and encodings (no Hungarian notation, no type prefixes).
- Use pronounceable, searchable names.
- Pick one word per concept and use it consistently throughout the codebase.
- Class names should be nouns; method names should be verbs.
